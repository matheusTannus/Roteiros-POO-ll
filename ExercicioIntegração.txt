1) Teste de integração é uma forma de vc testar diferentes partes ou todas as partes do código, para verificar se o sistema esta rodando corretamente, podendo ser durante a escrita do código ou até mesmo no final, garantindo que não acha erros.

2) Teste unitário verifica uma parte pequena do código isoladamente, já o Teste de integração verifica se várias partes do sistema funcionam juntas corretamente.

3) Não, mas quando realizado automatizado ele te traz mais velocidade com a mesma precisão de quando não automatizado, sendo mais fácil de usar com repetição.

4) Garantem que diferentes partes do sistema funcionem todas juntas corretamente, ajudando a identificar erros na comunicação entre módulos.

5) - Erro de comunicação entre módulos, quando uma parte do sistema não envia ou recebe dados corretamente da outra parte, causando falhas.
- Dados incorretos, quando um módulo envia dados em um formato diferente do esperado pelo outro, gerando erros ou resultados incorretos.

6) - Identificação dos módulos: quais partes serão integradas e como se relacionam
- Definição dos cenários de teste: determinar funcionalidade e interação serão validas
- Criação dos casos de teste: estabelecer dados de entrada e resultados para cada cenário
- Execução dos testes: aplicar os testes e monitorar comportamento do sistema 
-  Validação dos testes: comparar os resultados obtidos com o esperado
- Documentação: registrar o testes realizados, resultados e eventuais problema encontrado

7) - Integração vertical: testa a comunicação entre camadas 
- Integração horizontal: testa a comunicação entre módulos/serviços do mesmo nível
